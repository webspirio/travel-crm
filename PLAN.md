# AnyTour CRM — MVP Implementation Plan

## Context

Building a frontend-only MVP demo of a travel CRM for AnyTour — a German bus tour company running trips to Italian coastal resorts. The client needs a **live shareable URL** ASAP. Backend will be Supabase (later); this phase is pure frontend with mock data.

The project uses a devcontainer workflow — all development (npm install, shadcn init, coding) happens **inside the container**. Phase 0 sets up the devcontainer and minimal files needed to open it. Everything else happens after `devcontainer up`.

Reference project: `webspirio-presentation-2026-04` — reuse devcontainer, deploy workflow, configs.

## Phase 0: DevContainer + Minimal Scaffold (run on host) -- DONE

This phase creates ONLY the files needed to open the devcontainer. All npm/shadcn/build steps happen inside it.

### Files created (9 files):

- `.devcontainer/devcontainer.json` — Node 22, Chromium, port 5173, AI tools mounts, persistent bash history
- `.devcontainer/post-create.sh` — auto-scaffolds Vite + React 19 + Tailwind v4 + shadcn/ui + all deps on first container build
- `.devcontainer/post-start.sh` — status display on container start
- `.github/workflows/deploy.yml` — two-job GitHub Pages deploy (build -> deploy)
- `.gitignore` — standard Vite + Claude Code ignores
- `vite.config.ts` — React + Tailwind plugins, base `/travel-crm/`, path alias `@/`
- `tsconfig.json` — root config with `@/*` path alias
- `tsconfig.app.json` — ES2023, strict, react-jsx, bundler resolution
- `tsconfig.node.json` — for vite.config.ts only

### How to start:

1. Open in VS Code / Cursor
2. "Reopen in Container"
3. `post-create.sh` auto-runs and scaffolds everything
4. `npm run dev` to start

## Phase 1: Core Infrastructure — inside devcontainer (~60 min)

> Everything below runs INSIDE the devcontainer.

### 1.1 Update configs generated by scaffold

After `post-create.sh` runs, shadcn init will have created `components.json` and modified `src/index.css`. We need to:
- Verify `components.json` has `"style": "radix-nova"`, add reactbits registries
- Update `src/index.css` to use CRM-appropriate colors (neutral palette with blue/indigo primary)
- Create `eslint.config.js` (flat config from reference)

### 1.2 Type definitions — `src/types/index.ts`

All domain types:
- `BusType = '55' | '79'`
- `SeatStatus = 'free' | 'selected' | 'reserved' | 'sold' | 'blocked'`
- `TripStatus = 'planned' | 'booking' | 'confirmed' | 'in-progress' | 'completed' | 'cancelled'`
- `RoomType = 'single' | 'double' | 'triple' | 'family'`
- Interfaces: `Client`, `Trip`, `Hotel`, `Room`, `HotelBlock`, `Booking`, `Manager`
- `SeatCell = { type: 'seat'; id; number; status; price?; passengerName? }`
- `SpecialCell = { type: 'special'; kind: 'driver' | 'door' | 'toilet' | 'stairs' | 'kitchen' }`
- `Cell = SeatCell | SpecialCell | null`
- `DashboardStats`

### 1.3 Mock data generators — `src/data/`

All use `@faker-js/faker` v10, `seed(42)`, `refDate = new Date('2026-06-01')` for determinism.

| File | Content |
|------|---------|
| `managers.ts` | 3 managers + 1 owner (hardcoded) |
| `clients.ts` | 50+ clients — 30 Ukrainian locale, 20 German locale |
| `hotels.ts` | 8-10 Italian coastal hotels with star ratings + room blocks |
| `seats.ts` | Layout generators for 55-seat and 79-seat buses |
| `trips.ts` | 20 trips (6 Italian destinations x 4 German/Czech origins) |
| `bookings.ts` | ~80 bookings linking clients -> trips -> seats -> hotels |
| `stats.ts` | Compute dashboard KPIs from generated data |
| `index.ts` | Generate ONCE at module level, export as constants |

**Critical:** Data is generated ONCE and exported as module-level constants. No re-generation on render.

### 1.4 i18n — `src/i18n/`

| File | Content |
|------|---------|
| `config.ts` | i18next init, LanguageDetector, fallbackLng `'uk'`, 6 namespaces |
| `i18next.d.ts` | TypeScript augmentation for type-safe `t()` |
| `locales/uk/common.json` | Nav labels, common UI terms (Ukrainian) |
| `locales/de/common.json` | Same in German |
| `locales/uk/dashboard.json` | Dashboard strings |
| `locales/de/dashboard.json` | Dashboard strings |

Start with `common` + `dashboard` only. Add others per screen.
Ukrainian plurals: `_one` / `_few` / `_many` / `_other` (i18next handles CLDR).

### 1.5 Utilities — `src/lib/format.ts`

- `formatCurrency(value, locale)` — `Intl.NumberFormat` with EUR
- `formatDate(date, locale)` — `Intl.DateTimeFormat`
- `formatDateRange(start, end, locale)`

### 1.6 Config — `src/config/`

- `site.ts` — `{ name: "AnyTour CRM", description: "..." }`
- `nav.ts` — navigation items: `[{ titleKey, path, icon }]` for Dashboard, Trips, Clients, New Booking

### 1.7 Layout — `src/components/layout/`

### 1.10 Page stubs (5 files)

Minimal components for: dashboard, trips/list, trips/detail, clients/list, bookings/new.

## Phase 3: Trips List + Trip Details + SeatMap (~90 min)

### 3.1 Reusable DataTable — `src/components/data-table/`

| File | Purpose |
|------|---------|
| `data-table.tsx` | Generic wrapper: `columns` + `data` -> shadcn `Table` |
| `data-table-pagination.tsx` | Prev/next + page size select |
| `data-table-toolbar.tsx` | Search input + filter slot |
| `data-table-column-header.tsx` | Sortable headers with arrows |

### 3.2 Trips List — `src/pages/trips/list.tsx`

Columns: name (link), origin->destination, departure, bus type (`Badge`), occupancy (`Progress`), status (`Badge`), price.
Toolbar: destination filter, status filter, search.
i18n: `uk/trips.json` + `de/trips.json`.

### 3.3 SeatMap — `src/components/bus/`

| File | Purpose |
|------|---------|
| `seat-map.tsx` | Container: CSS Grid for 55, `Tabs` for 79 (upper/lower) |
| `seat-cell.tsx` | Seat `Button` with status variant + `Tooltip` |
| `seat-legend.tsx` | Color legend for 5 statuses |

**55-seat layout:**
- Row 0: `[driver, null, null, null, door]`
- Rows 1-13: `[seat, seat, null, seat, seat]` (null = aisle)
- Row 14: `[seat, seat, seat, null, toilet]`

**79-seat (double-decker):**
- Upper: 15 rows x (2+aisle+2) = 60 seats
- Lower: kitchen/WC + 2 face-to-face tables + ~19 seats
- shadcn `Tabs` "Upper deck" / "Lower deck"

Status -> variant: free=outline, selected=default, reserved=secondary, sold=muted, blocked=ghost+strikethrough.
ARIA: `role="grid"`, keyboard nav, `aria-label` with seat number + status.

### 3.4 Hotel cards — `src/components/hotel/`

- `hotel-card.tsx` — name, stars, photo placeholder, room breakdown
- `room-progress.tsx` — room type label + `Progress` + "12/20"

### 3.5 Trip Details — `src/pages/trips/detail.tsx`

Header with trip info. shadcn `Tabs`:
- **Bus** — SeatMap + seat stats
- **Hotels** — grid of HotelCards
- **Clients** — DataTable of passengers

**Phase 3 result:** Trips list -> click -> interactive seat map + hotel cards.

## Phase 4: Booking Form + Clients (~90 min)

### 4.1 Zustand store — `src/stores/booking-store.ts`

`BookingDraft` with `persist` middleware -> localStorage `'anytour-booking-draft'`.

### 4.2 Booking form — `src/components/booking-form/`

| File | Step |
|------|------|
| `stepper.tsx` | Horizontal 6-step indicator |
| `step-client.tsx` | 1: Client search/create (shadcn `Command`) |
| `step-trip.tsx` | 2: Trip selection |
| `step-seat.tsx` | 3: SeatMap (reused) |
| `step-hotel.tsx` | 4: Hotel + room type |
| `step-pricing.tsx` | 5: Price calc + commission |
| `step-summary.tsx` | 6: Summary + confirm -> `Sonner` toast |

### 4.3 Validation — `src/lib/validators.ts`

Zod v4 schemas per step. `@hookform/resolvers` v5 supports zod v4.

### 4.4 Booking page — `src/pages/bookings/new.tsx`

Orchestrates stepper + steps. Previous/Next/Submit navigation. Zustand store.

### 4.5 Clients List — `src/pages/clients/list.tsx`

DataTable with: `Avatar` (initials), name, nationality, email, phone, bookings count, last trip.
Row click -> shadcn `Sheet` slide-over with client details + booking history.
i18n: `uk/clients.json` + `de/clients.json`.

**Phase 4 result:** Complete booking wizard + clients list. All 5 screens functional.
